# Промт для старта сессии (копируй целиком в начало каждой новой сессии)

```
Ты работаешь над проектом RemoteDeskPBX (C:\RemoteDeskPBX). Правила сессии:

1. СНАЧАЛА прочитай: AGENTS.md, BACKLOG.md, STATE.md, CLAUDE.md (если есть).
   Возьми ОДНУ задачу из BACKLOG.md (я скажу какую; если нет — предложи по приоритету P0>P1>P2).
2. Одна сессия = одна задача. Не хватайся за несколько. Мелкие уточнения спрашивай сразу.
3. КРИТИЧНО про деплой сервера: приложение и сервер лежат в РАЗНЫХ репозиториях.
   - Разработка: GitHub Ilyaworks/RemoteDeskPBX (ветка master) — тут весь код.
   - Render деплоит НЕ отсюда, а из Ilyaworks/remotedeskpbx-server (ветка main),
     командой npm start -> node server.js.
   - Чтобы изменения сервера попали на прод: правь C:\RemoteDeskPBX\server.js,
     затем скопируй его в клон remotedeskpbx-server и запушь в main.
   - Проверка живого прода: GET https://remotedeskpbx-server.onrender.com/ должен
     содержать "server":"express-v2". /admin — пароль admin123.
4. Два приложения (Electron): Client (хост, делится экраном+звуком) и Employee
   (сотрудник, смотрит и управляет). Исходники: src/client, src/employee, src/main.
   Сборка: npm run pack:client / pack:employee (portable .exe в release/).
   Если менял код приложения — обязательно пересобери соответствующий .exe.
5. Тесты обязательны перед коммитом:
   - server.js: node --check server.js; для логики — запусти локально (PORT=39xx
     node server.js) и проверь curl-ом; админку проверь в браузере (вход admin123).
   - Приложения: npm run build (tsc+webpack) без ошибок.
6. После задачи: коммит с понятным сообщением (Co-Authored-By: Claude ...),
   обнови статус в BACKLOG.md (todo->done), допиши хэндофф в STATE.md
   (что сделано, что осталось, как проверить).
7. Грабли (не наступай повторно, см. CLAUDE.md):
   - Админ-HTML отдаётся через res.send(`...`) — это template literal. Экранированные
     одинарные кавычки в onclick ВНУТРИ строк должны быть \\' (двойной бэкслеш),
     иначе \' схлопнется в ' и сломает весь скрипт (белый экран).
   - Контракт сигналинга: /join возвращает {type:'ok'} (клиент это проверяет).
   - render-server/ в этом репо gitignore'ится и НЕ деплоится — не путать с прод-репо.

Задача на эту сессию: <ВСТАВЬ НОМЕР, например T7>
```
