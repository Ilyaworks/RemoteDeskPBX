### T1 — Сохранение пароля сотрудника / вход один раз · P1 · done
**Сделано:** «Запомнить меня» (по умолчанию вкл.), авто-вход при запуске, кнопка «Выйти» очищает. Учётка шифруется Electron `safeStorage` и лежит в `userData/employee-creds.bin` (fallback без шифрования с префиксом `PLAIN:`). IPC `creds-save/load/clear` в `employee.ts`, логика в `employee/App.tsx`. Требует пересборки employee.exe.

### T2 — Чат: плавающее окно у клиента + снизу у сотрудника · P1 · done
**Сделано:** у клиента — отдельное frameless always-on-top `BrowserWindow` (`resources/chat.html`) справа снизу рабочего стола, всплывает при сообщении сотрудника; встроенный чат из окна клиента убран. Мост: входящее → IPC `chat-show` → окно; ввод в окне → `chat-send` → окно приложения → DataChannel `chat`. При отключении окно скрывается. У сотрудника чат перенесён вниз основного окна во всю ширину. Требует пересборки обоих .exe.

### T6 — Вставка ID: вставляется 6 цифр вместо 9 · P0 · done
**Диагноз:** у поля `maxLength={9}` (`src/employee/App.tsx:311`) обрезает **сырую** вставку (с пробелами/форматированием, напр. «111 222 333») ДО удаления нечисловых символов в `onChange` — часть цифр теряется.
**Реализация:** убрать `maxLength`, оставить только `onChange`: `value.replace(/\D/g,'').slice(0,9)`.
**Готово когда:** вставка «111 222 333» и «111222333» даёт ровно 9 цифр.

### T7 — Скриншоты сотрудника → в Документы/папку приложения · P1 · done
**Сделано:** IPC `save-screenshot` в `employee.ts` пишет каждый принятый снимок в `Документы/RemoteDeskPBX/<код>/<timestamp>.jpg` (код санитизируется, fallback `general`). UI показывает путь + оставлена ссылка «Скачать вручную».
**Фикс после теста:** запрос скриншота уходил по каналу `control`, а клиент слушает `screenshot` → снимок терялся и папка не создавалась. Исправлена маршрутизация (сотрудник шлёт по `screenshot`), клиент переиспользует поток без повторного выбора экрана. Путь пишется через `app.getPath('documents')` — учесть OneDrive-редирект (точный путь виден в логе сотрудника).

### T8 — Авто-скриншоты каждые 30с не сохраняются на сервере · P0 · done
**Сделано:** `/register` теперь возвращает `sessionId`; клиент снимает кадр из уже захваченного `localStream` (без повторного `getDisplayMedia`) и шлёт `POST /screenshot` с `sessionId` каждые 30с (первый — через 3с). Таймер стартует после register, останавливается в `cleanup`. Проверено локально curl: register→sessionId, screenshot→файл в `screenshots/<sessionId>/`. Требует деплоя server.js на прод (repo `remotedeskpbx-server`) + пересборки client.exe.
**Фикс после теста:** захват сделан надёжным (`grabFrameDataUrl`: video-элемент + fallback `ImageCapture`) + логи в панель клиента.
**Ограничение (Render):** диск эфемерный — снимки пропадут при редеплое. Для постоянного хранения нужен внешний storage (не сделано, обсудить).

### T9 — Админка: индикатор активной вкладки + переход по сессиям сотрудника · P1 · done
**Было:** `switchTab`/`loadData` перерисовывали только `#tab-content`, класс `.active` оставался на «Сотрудники». Клик по сессиям сотрудника не переключал вкладку.
**Фикс:** `switchTab`/`loadData` вызывают `showDashboard()` (пересборка вкладок с active); `filterByEmployee` ставит `currentTab='sessions'`. Задеплоено на прод.

### T10 — Сессия не отобразилась у сотрудника «Егор» · P0 · done
**Было:** `employeeName` у сессии нигде не проставлялся (`/register`=null, `/join` не писал).
**Фикс:** сотрудник шлёт `employeeLogin`/`employeeName` при `/join`; сервер пишет их в открытую сессию по `code`; админка считает и фильтрует сессии по логину. Проверено curl: register→join→сессия с employee/employeeName, фильтр по логину = 1. Требует пересборки employee.exe + деплой сервера.

### T11 — Админка: кнопка «Выйти» + не разлогинивать при F5 · P1 · done
**Сделано:** токен админа хранится в `sessionStorage` (`rdpbx_admin_token`), при загрузке `initAuth()` восстанавливает сессию (обновление страницы не выкидывает); кнопка «Выйти» в шапке чистит токен и возвращает на логин; смена пароля обновляет сохранённый токен. Проверено в браузере, задеплоено на прод.
