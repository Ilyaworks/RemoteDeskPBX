## Задачи-фичи и баги

### T1 — Сохранение пароля сотрудника / вход один раз · P1 · todo
**Сейчас:** авторизация каждый запуск, нет сохранения (`src/employee/App.tsx`, `src/main/employee.ts` — нет `localStorage`/store).
**Надо:** «Запомнить меня» + авто-вход при следующем запуске; кнопка «Выйти» очищает.
**Реализация:** хранить учётку в `app.getPath('userData')` через Electron `safeStorage` (шифрование) + IPC. Не `localStorage` (сбросится).
**Готово когда:** после входа с «запомнить» повторный запуск сразу открывает рабочий экран.

### T2 — Чат: отдельное окно + снизу · P1 · todo
**Сейчас:** чат встроен снизу (`chatMessages`/`chatInput`).
**Надо:** по умолчанию чат снизу; когда приходит новое сообщение (на стороне клиента — когда сотрудник написал) — всплывает **отдельное окно** чата.
**Реализация:** второе `BrowserWindow` для чата, синхронизация через IPC + DataChannel `chat`.
**Готово когда:** клиент получает всплывающее окно чата на новое сообщение; у сотрудника чат снизу.
**❓ Уточнить:** отдельное окно нужно у клиента, у сотрудника, или у обоих?

### T3 — Убрать меню браузера сверху (чистое окно) · P0 · todo

**Готово когда:** ни в client, ни в employee нет верхнего меню.

### T6 — Вставка ID: вставляется 6 цифр вместо 9 · P0 · todo
**Диагноз:** у поля `maxLength={9}` (`src/employee/App.tsx:311`) обрезает **сырую** вставку (с пробелами/форматированием, напр. «111 222 333») ДО удаления нечисловых символов в `onChange` — часть цифр теряется.
**Реализация:** убрать `maxLength`, оставить только `onChange`: `value.replace(/\D/g,'').slice(0,9)`.
**Готово когда:** вставка «111 222 333» и «111222333» даёт ровно 9 цифр.

### T7 — Скриншоты сотрудника → в Документы/папку приложения · P1 · todo
**Сейчас:** скриншот только как ссылка на скачивание (`src/employee/App.tsx:404`).
**Реализация:** IPC в main → запись JPEG в `app.getPath('documents')/RemoteDeskPBX/<код или дата>/`.
**Готово когда:** каждый снимок автоматически лежит в `Документы\RemoteDeskPBX\`.

### T8 — Авто-скриншоты каждые 30с не сохраняются на сервере · P0 · todo
**Диагноз:** фичи «авто-скриншот каждые 30с» **нет в коде вообще**. Есть только ручной P2P-снимок через DataChannel (`screenshot-request`/`screenshot-data`), который на сервер (`POST /screenshot`) не отправляется. Эндпоинт `/screenshot` на сервере есть, но его никто не вызывает.
**Реализация:** таймер 30с на стороне сотрудника (или клиента) → `POST /screenshot` c правильным `sessionId` (связать с T10). Плюс учесть эфемерный диск Render (скриншоты пропадут при редеплое — возможно нужен внешний storage, обсудить).
**Готово когда:** в админке у сессии появляются снимки с шагом ~30с.

### T9 — Админка: плашка под «Сотрудниками» не двигается · P1 · todo
**❓ Требуется уточнение:** какая именно плашка/элемент, что должно происходить (перетаскивание? сворачивание? прокрутка?). Пришли скриншот/описание — тогда поставлю диагноз.

### T10 — Сессия не отобразилась у сотрудника «Егор» · P0 · todo
**Диагноз:** `employeeName` у сессии **нигде не проставляется**. Сервер `/register` создаёт сессию с `employeeName: null`, `/join` его не заполняет, в клиенте/сотруднике кода установки нет. Поэтому фильтр админки по сотруднику пуст и сессия «ничья».
**Реализация:** при `/join` сотрудник шлёт свой `login/name`; сервер записывает его в сессию по `code`; админка начинает показывать.
**Готово когда:** после подключения Егора сессия видна в админке под Егором.
